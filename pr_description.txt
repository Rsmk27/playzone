🎯 **What:** The Hangman component lacked tests to verify its core game logic, including word selection, guessing letters, tracking score and incorrect guesses, and determining win/loss conditions.

📊 **Coverage:** This PR adds a complete test suite for `src/games/Hangman.jsx`. The tests cover:
- Initial state rendering (score, incorrect guesses, keyboard, new word button).
- Handling correct guesses (disabling button, revealing letter, not increasing incorrect guess count).
- Handling incorrect guesses (disabling button, increasing incorrect guess count, no letters revealed).
- Winning condition (guessing all letters in a predictable word by mocking `Math.random`, showing win banner, score increases, disabling keyboard).
- Losing condition (guessing 6 incorrect letters, showing loss banner with the word, score remains 0, disabling keyboard).

✨ **Result:** Test coverage for the Hangman game component logic is significantly improved, adding confidence and guarding against future regressions in the codebase.
